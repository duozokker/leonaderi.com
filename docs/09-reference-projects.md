# Reference Projects and Patterns

## Why These References
These projects informed architecture and production strategy for this repository.

## Reference List

### Phaser + React template (official)
- Repo: <https://github.com/phaserjs/template-react>
- Relevance:
  - clean React/Phaser bridge pattern
  - Vite-compatible setup
  - scene lifecycle examples

### Top-down Phaser template with React UI overlays
- Repo: <https://github.com/blopa/top-down-react-phaser-game-template>
- Relevance:
  - dialog system patterns
  - virtual gamepad concept for mobile
  - map + assets organization ideas

### 2D game portfolio template (Kaboom)
- Repo: <https://github.com/JSLegendDev/2d-portfolio-kaboom>
- Relevance:
  - proven "portfolio as game" concept
  - desktop/mobile control pattern
  - practical hosting/deployment baseline

### RPG-style portfolio example
- Repo: <https://github.com/alsocreative/portfolio-adventure>
- Relevance:
  - modular content-oriented approach
  - room-based discovery model

## How We Apply These Insights
- Use Phaser + React split for robust game mechanics plus flexible UI.
- Keep content in typed glossary files.
- Keep mobile controls and onboarding built-in from day one.
- Keep deployment static-hosting friendly.
